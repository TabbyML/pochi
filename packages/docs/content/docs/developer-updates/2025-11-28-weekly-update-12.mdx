# Weekly Update #12

### TL;DR

This weeks update is packed with improvements to task visibility, worktree workflows, and NES accuracy. We also made foundational changes to how tasks are stored, which sets us up for upcoming GitHub and multi-worktree features.

<Callout type="info" title="Breaking Change: Task list reset in v0.16.0">
  This release includes internal changes to task state management and storage. As a result, existing tasks from previous versions cannot be migrated, and your task list will be cleared after upgrading to v0.16.0.
No action is needed as new tasks will behave normally going forward.
</Callout>

- **Tasks grouped by worktree:** Pochi automatically groups tasks by their associated Git worktree with active worktrees at the top and deleted worktrees collapsed into a dedicated section. Each worktree shows its own tasks, status, and metadata which makes it much easier to manage parallel workstreams and multiple agents.

- **Worktree-aware task tabs:** Pochi now opens tasks from the same Git worktree in the same VS Code tab group, and tasks from different worktrees in separate tab groups. This keeps parallel branches visually separated while you work.
